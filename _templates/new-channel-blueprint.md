# New Channel Blueprint — TEMPLATE

Copy this file to `<NewChannelFolder>\channel.md` and fill it in to launch a new channel/category.
The global `..\..\CLAUDE.md` rules (Prime Directive, pipeline, standards) apply unchanged — this
file only defines the new channel's **identity and strategy**.

**Recommended way to fill it:** have Claude run a research pass first (niche viability, competition,
RPM, positioning) — the same kind that produced the `History` channel. Prompt:
> "Research whether `<niche>` is a viable faceless-channel lane in the current format: who already
> owns it, the supply gap, realistic RPM, the best positioning angle, and 10 launch ideas. Verify
> high-stakes claims (RPM, monetization policy changes) against current sources."

Delete the _italic guidance_ as you complete each section.

---

## 0. Pre-launch viability gate (answer before committing)

- [ ] **Supply gap:** is there an under-served angle, or is it saturated by big channels?
      _Specificity beats breadth — pick the gap, not the broad category._
- [ ] **Topic supply:** can you make 50+ videos without repeating yourself?
- [ ] **Audience value:** does it skew Tier-1 / educated (higher RPM)? Who advertises to them?
- [ ] **Format fit:** does it work as whiteboard/stick-figure + narration?
- [ ] **Creator fit:** will you still care about it after 30 videos? _(This is the one most people fail.)_
- [ ] **Compliance fit:** can each video carry a genuine original angle + fact-checking? _(Prime Directive.)_

If two or more are weak, pick a different lane.
